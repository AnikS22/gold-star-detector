# gold-star-detector — MidasMap slide deck (React)

Interactive **MidasMap** presentation with Framer Motion transitions. Content and figures are aligned with the parent **[MidasMap](https://github.com/AnikS22/MidasMap)** repo (`docs/MidasMap_Presentation.md`, `docs/Final_Presentation_Slides.md`, evaluated metrics).

## Run locally

```bash
cd gold-star-detector
npm install
npm run dev
```

Open the URL Vite prints (usually `http://localhost:8080`). **← →** or **Space** to advance; **F** fullscreen.

## Extra images (`media/`)

Bundled **presentation PNGs**, **deck JPEGs**, and **real unlabeled** TEM crops (from the same micrographs as the deck). See [`media/README.md`](media/README.md). Refresh crops after changing `src/assets/`: `python media/scripts/export_unlabeled_from_assets.py`.

## MidasMap figures (`public/midasmap-figures/`)

PNG panels are copied from MidasMap `presentation_figures/` (generated by `scripts/generate_presentation_figures.py`). To refresh with **real** TEM + model overlays:

```bash
cd ../   # MidasMap repo root
export MIDASMAP_DATA_ROOT="/path/to/Gold Particle Labelling"
python scripts/generate_presentation_figures.py --checkpoint checkpoints/final/final_model.pth --synapse S27
cp presentation_figures/*.png gold-star-detector/public/midasmap-figures/
```

Then rebuild or refresh the dev server.

## Accuracy notes

- **LOOCV mean F1 0.943** = mean over **8** folds with enough labels (S7/S15 excluded).
- **Per-class 6 nm / 12 nm F1 and 100% recall** = example **S1-fold** table in MidasMap docs, not averaged across all folds.
- **1790 px/µm** = FFRIL calibration used to fix micron coordinate bugs.
- **~24.4M parameters**, **~10 s/image GPU** inference = as documented in MidasMap README / presentation doc.
